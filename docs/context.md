# Context: 인스톨러 개선 및 범용성 확장

## 배경
- 사용자가 여러 프로젝트에서 본 플러그인을 설치할 때 설치 경로가 제각각인 문제를 보고함.
- 특정 환경에서 `start.bat` 실행 시 `.venv`가 이미 존재함에도 불구하고 에러를 출력하며 중단되는 현상 발생.
- **서브모듈 설치 시 부모 프로젝트의 기술 스택을 제대로 탐지하지 못하는 한계 발견.**

## 현재 상태
- **완료:** 정적 설치 경로(`.agents/tech-stack-organizer`) 강제 및 체크 로직 구현.
- **완료:** `start.bat`에 `uv` 기반의 강인한 가상환경 재사용 및 자동 복구 로직 적용.
- **완료:** `discover-stack.py`에 재귀적 상위 루트 탐지 및 `pyproject.toml` 파싱 엔진 탑재.

## 주요 결정 사항
- **표준 경로(SSOT Path):** `.agents/tech-stack-organizer`
- **가상환경 전략:** `uv`를 사용하되, 기존 `.venv` 내부 무결성을 검증하여 재사용.
- **범용 탐지 엔진:** 서브모듈의 `.git` 파일 유무와 관계없이 부모 프로젝트의 루트를 찾아 `deep scan` 수행.
